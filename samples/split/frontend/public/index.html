Split Payments - Information

This is a Demo to demonstrate Split Payments. Split Payments can be carried out via both the Card API and Account API.
Go here to understand how split payments work and how to create subaccounts

  • In this demo, the amount being charged is NGN5000.
  • The amount is split like so:
  • First Subaccount gets NGN1500
  • Second Subaccount gets NGN2800
  • The remaining money goes to the merchant's Rave balance

Please Note That: The demo card below is a Mastercard and hence this demo does not demonstrate how to handle VISA cards. Kindly see the CLI documentation on how to handle VISA cards
Card No: 5399838383838381 | CVV: 470 | PIN: 3310 | Expiry: 10/22 | OTP: 12345

Kindly use the account number below.
Account Number: 0690000031 (Access Bank) | OTP: 12345

Please note that the "Account Name" field has been disabled. We dynamically fetch the account's name using Rave's account resolve API: https://ravesandbox.azurewebsites.net/flwv3-pug/getpaidx/api/resolve_account

In general, we highly recommend that you check out the codebase generated for you to see and understand how split payments work.
You definitely can reuse any parts of it that you want


We recommend that you edit the code base to use your own sandbox keys and sub account IDs to better understand how money is split between subaccounts
